> 🧐 배경: IntegerBox, StringBox 처럼 여러개의 box를 만들 때 xxxBox등 여러개를 만들려면 반복되는 불필요한 작업이 필요하다. 따라서 이를 해결하기 위해서 Generic 을 사용하여 해결할 수 있다. 
> 
> 하지만 제네릭이 어떻게 만들어졌는지 순서대로 구현과정을 경험해보자. 

## 다형성을 활용한 중복 해결 시도
다형성 덕분에 코드의 중복을 제거하고, 기존 코드를 사용할 수 있지만 입력 시 실수로 원하지 않는 데이터 타입이 들어가는 `타입 안정성` 문제가 발생할 수 있다. 

또한 반환 시점에서 `Object`를 반환하기 때문에 원하는 데이터 타입을 <mark style="background: #FFB8EBA6;">정확히 받을 수 없고 이로 인해 다운 캐스팅</mark>으로 인해 오류가 발생할 수 있는 문제가 발생한다. 

## 제네릭을 이용한 해결
> 반복되는 작업을 줄이고 동시에 다형성을 이용했을 때 발생할 수 있는 타입 안전성의 문제를 해결할 수 있다. 
> 

```java
public class GenericBox<T> {
	...
}
```
<> 다이아몬드 기호에 `T` 를 넣어서 선언한다. 
<Br>
자바 컴파일러가 입력한 타입 정보를 기반으로 이러한 코드가 있다고 가정하고 컴파일 과정에서 타입 정보를 반영한다.

타입 추론 : 생성하는 제네릭 타입 생략 가능
```java
	genericBox<String> stringBox = new Generic<String>(); 
	genericBox<String> stringBox = new Generic<>(); // 생략가능 
```

## 📍 제네릭 핵심
> 사용할 타입을 미리 결정하지 않는다, 실제 사용 시점에서 타입을 결정하는 것
> 해당 클래스를 실제 사용하는 생성 시점에 내부에서 사용할 타입을 결정하는 것

### 메서드의 매개변수를 통한 정리
- 메서드는 매개변수에 인자를 전달해서 사용할 값을 전달
- 제네릭은 타입 매개 변수(=`T`) 에 타입 인자를 전달하여 사용할 값을 전달

### Generic Type (제네릭 타입)
- Generic 은 일반적인, 범용적인 이라는 뜻
- 클래스나 인터페이스를 정의할 때 타입 매개변수를 사용하는 것
- 제네릭 클래스, 제네릭 인터페이스 모두 합쳐서 `Generic Type`

### 제네릭 명명 관례
일반적으로 대문자
용도의 맞는 단어의 첫글자를 사용
- E - Element
- K - Key
- N - Number
- T - Type
- V - Value
- S,U,V etc. -2nd, 3rd, 4th, types

### row type
> 🧐 제네릭 타입을 사용할 때 `<>`를 사용하지 않아도 되는데 이를 row type 또는 원시타입이라 한다. 
> 하지만 제네릭을 사용할 때는 항상 `<>` 사용하는 시점에 지정하는게 좋다. 
> 
> why❓ : 이전 버전의 호환성 문제로 남겨둔 문법이라 사용 안하는게 좋음

## 제네릭 활용 시 문제점
> 제네릭 타입을 사용하여 클래스를 정의하는 경우에는 메서드를 정의하여 사용할 때는 컴파일 시점에서 데이터 타입이 정해지기 때문에 오류가 발생한다


따라서 `toString()`,  `equals()` 의  `object`타입의 기능만 사용할 수 있다. 

### 타입 매개변수 제한

`AnimalHospital<T>` 를 사용하는 경우에는 제네릭에 `Integer`, `String` 같은 무분별하게 클래스를 받는게 아니라 Animal 에 해당하는 클래스만 받고싶어한다.
<br>
따라서 T인 제네릭이 Animal 에 해당하는 클래스나 그 자식을 받기 위해서 
`AnimalHospital<T extends Animal>` 로 받으면 컴파일러는 T 가 입력될 수 있는 값을 예측할 수 있다. 따라서 Animal 에 해당하는 메서드들도 컴파일러 시점에 예측하여 사용할 수 있다. 


## 제네릭 메서드
>제네릭 타입과 제네릭 메서드는 다른 기능이다. 

- 제네릭 타입 :  객체를 생성하는 시점에 타입인자를 전달함
- 제네릭 메서드 : 메서드를 호출하는 시점에서 전달함 

메서드에서만 한정적으로 제네릭을 사용하는 기능이다 
```
public static <T> T genericMethod(T t){} ...
```
정의 시 메서드의 반환 타입 왼쪽에 다이아몬드를 사용하여 `<T>` 와 같이 타입 매개변수를 적어준다. 

### 타입인자 명시적 전달
```
Integer value = GenericMethod.<Integer>numberMethod(10);
```

✏️ 제네릭 메서드는 호출 시점에 `<>` 을 사용하여 `<Integer>`와 같이 타입을 정하고 호출한다. (매개변수에서 어떤 변수가 들어오는지 타입 추론이 가능하기 때문에 <>은 생략가능)

## 제네릭 타입과 제네릭 메서드의 우선순위
> 제네릭 메서드가 높은 우선순위를 가진다.
