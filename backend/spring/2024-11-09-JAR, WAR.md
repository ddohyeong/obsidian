> 🧑🏻‍💻 Java 기반의 애플리케이션을 패키징 시 늘 듣는 두가지 JAR, WAR이 뭐고 차이점을 알아보자. 

# 1. JAR (JAVA Archive)
> JAR 파일은 클래스 파일, 리소스, 메타데이터 등 여러 Java 관련 파일을 포함하는 압축된 아카이브 형식. 이를 통해서 개발자는 보다 쉽게 배포할 수 있도록 Java 애플리케이션, 라이브러리 또는 모듈을 단일로 묶을 수 있도록 한다.  `zip`형식을 사용하여 압축되어 있다. 

## 어떻게 생성될까? 
- JDK(Java Development Kit) 에서 제공되는 jar 명령줄 도구를 사용하여 생성한다.
- 개발자는 선택적 매니페스트 및 보안 정보와 함께 JAR에 포함된 파일 및 디렉터리를 지정할 수 있다. 

### 매니페스트 파일의 목적은? 
JAR 및 해당 콘텐츠에 대한 메타데이터를 포함하는 JAR 파일의 선택적 구성 요소로 실행할 기본 클래스, 클래스 경로 종속성, 애플리케이션 버전등 정보를 제공. JAR내의 META-INF 디렉토리에 위치, JAR 을 실행 시 JVM 에서 읽음

# 2. WAR (Web Application Archive)
> 웹 어플리케이션을 패키징하기 위한 파일 포맷. Java 클래스, JSP, Servlet, XML, 라이브러리, 정적 웹 리소스(HTML, CSS) 등 웹 어플리케이션 구성 요소를 포함
> Java 의 JAR 옵션을 이용해 생성하는 JAR 파일의 일종이다. 

# 3. JAR, WAR 의 공통점은? 
> 모두 Java 애플리케이션의 배포 및 실행을 위한 파일로 `라이브러리, 클래스, 리소스 파일`을 포함하고, Java 실행환경에서 실행할 수 있다. 

1. Java 애플리케이션의 배포 형식
- 둘다 java 애플리케이션을 패키징하고 배포가능. java 클래스 파일, 라이브러리, <span style='color:var(--mk-color-red)'>설정 파일을 포함으로 실행 가능한 형태로 묶는 역할</span>

2. Java 압축 형식
- zip 형식으로 압축, 내부 구조는 ZIP 파일과 유사

3. 클래스 및 라이브러리 파일 포함 가능
- 둘다 `.class`파일과 `.jar`파일(라이브러리)을 포함 가능. 애플리케이션에서 사용할 모든 java 코드와 라이브러리를 패키징 할 수 있다. 

4. META-INF 디렉토리
- 이 안에는 메타데이터 파일이 들어있다. `MANIFEST.MF` 파일은 애플리케이션 엔트리 포인트, 버전 정보등을 정의하는 메타 데이터가 포함됨

# 4. JAR, WAR 차이점은?

| **구분**      | **JAR 파일**                                                                    | **WAR 파일**                                                                                  |
| ----------- | ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| **용도와 목적**  | - 독립 실행형 애플리케이션 패키징<br> - Java SE 애플리케이션에 적합<br> - Spring Boot로 내장 웹 서버 포함 가능 | - 웹 애플리케이션 패키징<br> - Java EE 애플리케이션에 적합<br> - 외부 웹 애플리케이션 서버에서 실행                           |
| **디렉토리 구조** | - `.class` 파일과 라이브러리 포함<br> - `META-INF` 디렉토리에 메타데이터 (`MANIFEST.MF`) 포함       | - `WEB-INF` 폴더에 `web.xml` 설정 파일 포함<br> - `WEB-INF/lib`에 라이브러리, `WEB-INF/classes`에 클래스 파일 배치 |
| **배포 방식**   | - 독립적으로 실행 가능 (`java -jar <파일명>.jar`)<br> - 외부 웹 서버 필요 없음                     | - 외부 웹 서버(Tomcat, JBoss 등)에 배포 필요<br> - 서버에서 여러 애플리케이션 동시에 관리 가능                            |
| **실행 환경**   | - 내장 웹 서버로 클라우드 및 컨테이너 환경에 적합<br> - 마이크로서비스 구조에서 주로 사용                        | - 외부 애플리케이션 서버의 기능(세션 관리, 로드 밸런싱 등)을 활용 가능<br> - 대규모 엔터프라이즈 환경에 적합                          |
| **서버 의존성**  | - 독립 실행 가능, 서버 의존성 없음                                                         | - 외부 애플리케이션 서버 의존                                                                           |
