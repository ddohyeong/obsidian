## 시간 복잡도
>  시간복잡도는 프로그램이 입력 크기에 대해 얼마나 빠르게 실행되는지 나타냄
>  보통 빅오 표기법을 의미한다. 

파이썬 기준으로 `1초 = 1억` 연산

따라서 풀이의 연산이 1억번 이하의 연산이 나오도록 해야한다. 

**정렬 알고리즘**을 통해서 시간복잡도에 대한 차이를 이해하도록 하자.

- O(N^2) 시간복잡도의 정렬 : 선택정렬, 삽입정렬, 버블정렬
- O(NlogN) 시간복잡도 정렬 : 퀵, 병합

> 🧐N의 크기에 따라서 계산하는 방법을 알아보자

**N = 10000 일때** 
- N^2 = 100,000,000 (1억번)
- NlogN = 10000 * log10000(=14) = 140,000 (14만번)
> 🧐 N이 10000일때 부터 N^2 은 아슬하게 1초 연산을 통과할 수 있을것이고, NlogN 복잡도를 가진 연산은 여유있게 통과할 수 있다. 

**N=100,000 일때** 
- N^2 = 100억
- NlogN = 100,000 * log100,000(=17) = 1,700,000 (170만번)

**N=1,000,000 일때**
- N^2 = 1조
- NlogN = 1,000,000 * log1,000,000(=20) = 20 * 1,000,000 (2000만번)

-> N의 크기가 10000이상으로 커지게 되면 N^2의 복잡도로는 시간초과가 발생하지만, NlogN이 경우에는 백만이 넘어가더라도 비교적 연산횟수가 낮은 것으로 알 수 있다. 

> 알고리즘을 설계할 때는 주어진 시간을 고려하여 적절한 시간복잡도의 연산횟수를 가지는 방법을 채택해야 한다.

## 공간 복잡도 정리
얼마나 많은 메모리를 사용하는지 나타냄

공간 복잡도가 큰 풀이는 문제를 풀경우 OOM(메모리 초과) 발생할 수 있다. 
그러나 엄청 큰 단위로 공간을 할당하는게 아니면 OOM이 발생하지 않는다.

**코테에서 공간복잡도 정리**
int = 4byte 기준으로 정리를 하겠다. 

1MB = 약 1,000,000Byte -> 25만개
128MB = 25만 x 128 = 32,000,000 (3천만개)

배열을 1000 * 1000 로 만들면 공간복잡도에서 메모리 초과가 발생할 수 있다.
